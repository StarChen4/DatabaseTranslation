# 军事数据库翻译原则（更新版）

## 数据来源说明

本翻译原则基于对数据库中14个表（DataAircraft, DataShip, DataSubmarine, DataFacility, DataGroundUnit, DataWeapon, DataSensor, DataMount, DataLoadout, DataComm, DataMagazine, DataWarhead, DataPropulsion）的390条样本数据的系统分析制定。

样本统计：
- 总样本数：390条
- 包含方括号的条目：127条（32.6%）
- 包含年份信息：42条（10.8%）
- 数量表达式：28条（7.2%）
- 北约代号：18条（4.6%）

---

## 原有翻译原则

### 1. 保留型号/编号
保留所有型号和编号，例如：
- "J-16" → "J-16"
- "F-11" → "F-11"
- "SH-60B" → "SH-60B"

### 2. 保留缩写，或在缩写后括号中加翻译
例如：
- "AAM" → "AAM(空对空导弹)"
- "LRASM" → "LRASM(远程反舰导弹)"

### 3. 保留项目代号/计划名称，或在后加括号中翻译注释
例如：
- "MIG-23BN Flogger-H" → "MIG-23BN Flogger-H(鞭笞者-H)"

### 4. 保留制造商名字，除非已有明确的中文名字对应
例如：
- "4x Waukesha Diesels" → "4台Waukesha柴油发动机"

### 5. 保留人名命名，但要翻译他的军衔，然后在后面加"号"
例如：
- "T-AK 3008 2nd Lt. John P. Bobo" → "T-AK 3008 约翰·P·博博少尉号"

### 6. 保留单位制
例如：mm, km, kg, kT, mT等

### 7. 翻译固定译名
例如：
- "Falcon" → "战隼"
- "Sidewinder" → "响尾蛇"
- "Seahawk" → "海鹰"
- "Phalanx" → "密集阵"

### 8. 翻译除上述说明外的英文单词
例如：
- "Single Rails" → "单联挂架"

### 9. Comments里若出现句子或段落则完整翻译

### 10. 当译名翻译不确定时，记录并保留原文

### 11. 注意军事术语的使用和翻译准确性、专业性，禁止文学化

## 补充翻译原则

### 12. 方括号内容的处理原则
**发现依据**：127条样本（32.6%）包含方括号

方括号通常用于表示：
- **替代型号**：保留所有型号信息
  - 示例：`OH-6D Cayuse [MD-500D, Model 369D]` → `OH-6D Cayuse [MD-500D, Model 369D]`
- **级别/类别**：翻译类别，保留型号
  - 示例：`LD 601 Tarlac [Makassar Class]` → `LD 601 塔拉克号 [Makassar级]`
- **技术规格**：翻译功能描述，保留技术型号
  - 示例：`127mm/64 Vulcano GLR IR [Guided Long Range]` → `127mm/64 Vulcano GLR IR [制导远程]`

**翻译原则**：保留方括号结构，根据内容类型分别处理

---

### 13. 北约代号（NATO Codename）的处理
**发现依据**：18条样本包含北约代号

常见北约代号包括：Flanker（侧卫）、Bear（熊）、Hind（雌鹿）、Flanker（侧卫）、Fishcan等

**翻译原则**：
- **有固定中文译名的**：保留原文，可在括号中添加中文译名
  - 示例：`Su-30MK2 Flanker G` → `Su-30MK2 Flanker G(侧卫G型)`
  - 示例：`Mi-24A Hind A` → `Mi-24A Hind A(雌鹿A型)`
- **无固定译名或译名不常用的**：保留原文
  - 示例：`J-7 Fishcan` → `J-7 Fishcan`

---

### 14. 口径/规格斜杠格式（mm/caliber）的处理
**发现依据**：18条样本包含此格式

格式通常为：`口径mm/倍径 型号 类型`

**翻译原则**：
- 保留口径和倍径数字
- 保留型号编号
- 翻译武器类型和功能描述
- 示例：
  - `127mm/64 Vulcano GLR IR [Guided Long Range]` → `127mm/64 Vulcano GLR IR [制导远程]`
  - `155mm/39 M109A4+ Self-Propelled Howitzer` → `155mm/39 M109A4+自行榴弹炮`

---

### 15. 中国装备代号的处理
**发现依据**：21条样本包含中国装备代号

包括：
- **飞机代号**：J-（歼）、JH-（歼轰）、H-（轰）、Y-（运）、Z-（直）、Q-（强）
- **导弹代号**：HQ-（红旗）、PL-（霹雳）、YJ-（鹰击）、CJ-（长剑）、DF-（东风）、CSS-（西方代号）
- **通用代号**：Type XXX（XX型）

**翻译原则**：
- 保留英文/拼音代号
- 可在首次出现时添加中文全称
- 示例：
  - `J-7L Fishcan D` → `J-7L（歼-7L）Fishcan D`
  - `Type 035A Ming` → `035A型明级`
  - `DF-21C` → `DF-21C（东风-21C）`

---

### 16. 俄罗斯装备代号的处理
**发现依据**：26条样本包含俄罗斯装备代号

包括：
- **飞机代号**：Su-（苏霍伊）、MiG-（米格）、Tu-（图波列夫）、Il-（伊留申）、An-（安东诺夫）、Mi-（米里）、Ka-（卡莫夫）
- **舰船代号**：Pr.（Project，项目）
- **武器代号**：SA-（地空导弹）、SS-（地地/舰舰导弹）、AT-（反坦克导弹）

**翻译原则**：
- 保留俄文拉丁转写代号
- 可在括号中添加中文厂商名
- 示例：
  - `Su-30MK2 Flanker G` → `Su-30MK2（苏-30MK2）Flanker G`
  - `Mi-24A Hind A` → `Mi-24A（米-24A）Hind A`
  - `Pr.956E Sovremenny III` → `956E型现代级III型`

---

### 17. 舰船命名规则
**发现依据**：14条样本采用标准舰船命名格式

格式通常为：`舰种代号 编号 舰名 [级别]`

**翻译原则**：
- 保留舰种代号（CVN, DDG, LST等）
- 保留编号
- 翻译舰名，在后加"号"
- 翻译级别为"XX级"
- 示例：
  - `CVN 65 Enterprise` → `CVN 65 企业号`
  - `LST 1179 Newport` → `LST 1179 纽波特号`
  - `LD 601 Tarlac [Makassar Class]` → `LD 601 塔拉克号 [Makassar级]`

---

### 18. 数量表达式的处理
**发现依据**：28条样本的Comments中包含数量表达

常见格式：`数字x`、`数字 pr Bty`、`数字 Cells`、`数字 rnds`

**翻译原则**：
- `数字x` → `数字台/座/门`（根据装备类型）
- `pr Bty` → `每连`
- `Cells` → `单元`
- `rnds` → `发`
- `Mounts` → `座`
- 示例：
  - `4x` → `4座`
  - `2x pr Bty` → `每连2座`
  - `[16 Cells]` → `[16单元]`

---

### 19. 年份和时间信息的处理
**发现依据**：42条样本的Comments中包含年份

**翻译原则**：
- 保留所有年份数字不翻译
- 示例：
  - `Comments: 1982, S-300PT-1` → `注释：1982年，S-300PT-1`
  - `Comments: 2015, Land and ASuW` → `注释：2015年，陆基和反舰`

### 21. 技术状态标识的处理
**发现依据**：6条样本包含状态标识

常见标识：
- `DEPRECATED` → 已弃用
- `Cancelled` → 已取消
- `Retired` → 已退役
- `Mod` / `Modified` → 改进型
- `Upgraded` → 升级型

**翻译原则**：
- 完整翻译状态标识
- 示例：
  - `Comments: DEPRECATED` → `注释：已弃用`
  - `Comments: Cancelled, Never Produced` → `注释：已取消，从未生产`
  - `TOW Retired` → `陶式导弹已退役`

### 22. 复合命名的优先级
**综合原则**

当一个条目同时包含多种元素时，按以下优先级处理：

1. **保留**：型号编号、制造商型号、技术规格数字
2. **可选翻译**：北约代号（如有固定译名）
3. **必须翻译**：功能描述、武器类型、状态标识
4. **结构保持**：方括号、圆括号、斜杠的原有结构

示例：
```
原文: 155mm/39 M109A4+ Self-Propelled Howitzer x 6
     ↓
译文: 155mm/39 M109A4+ 自行榴弹炮 x 6门
```

## 补充原则说明

### 补充缘由总结

1. **原则12-14（方括号、北约代号、口径格式）**：样本分析显示32.6%的条目包含方括号结构，需要明确处理规则以保持数据一致性。

2. **原则15-16（中俄装备代号）**：中国和俄罗斯装备各占约5-7%的样本，这些装备有特殊的命名系统，需要专门规则确保准确性。

3. **原则17（舰船命名）**：舰船命名有严格的军事规范，需要遵循"舰种+编号+舰名+号"的格式。

4. **原则18-19（数量和年份）**：Comments字段中频繁出现（合计占13%），需要统一翻译标准。

5. **原则20（缩写词典）**：建立标准缩写词典可以确保翻译的一致性和专业性，避免同一缩写出现多种翻译。

6. **原则21-22（状态标识和优先级）**：明确特殊标识的处理方式和复合情况下的优先级，确保翻译逻辑清晰。
